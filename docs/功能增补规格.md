# Freetrip 个人博客 · 功能增补规格说明书

## 文档目的与背景

本文是一份**功能增补规格（addition spec）**，面向一个**已经上线运行**的 Astro「翻页书」风格个人博客（站主「夜」/siyuan，品牌名 Freetrip）。文档不重写既有系统，只规定在现有架构之上**新增**的 7 项功能：从立刻该做的 SEO/元数据基建，到值得做的真实量化研究案例与双语 i18n。

每项功能都附带：精确到文件的改动清单、数据模型、可执行的实现步骤、边界情况、可勾选的验收标准与待站主拍板的待定问题。开发者可据此直接开分支动工。

所有改动遵循一条铁律：**不引入任何前端框架、不引入 StPageFlip、保持纯静态站点**。交互一律用 `is:inline` vanilla `<script>` + `window` 全局状态（跨 View Transition 持久化），延续现有 VinylPlayer / ThemeToggle / InkCanvas 的模式。

## 现状基线

> 以下为**已验证**的真实技术栈，规格中不得重复提议这些已存在的能力。

- **框架**：Astro 6.1.9，**纯静态站点**（SSG），`package.json` 依赖**只有** `astro` 一项。**无** React / Vue / 任何客户端框架，**无** StPageFlip。
- **「翻页书 / flipbook」是纸张质感的隐喻**，铺在**真实路由的 `.astro` 页面**之上，**不是**真正的翻页 canvas。导航 = 真实路由 + Astro `ClientRouter`（View Transitions 交叉淡入）。
- **交互模式**：`is:inline <script>` + `window` 全局状态（如 VinylPlayer 的 `window._vinylState`、ThemeToggle 的 `astro:before-swap` / `astro:after-swap` 钩子），跨 View Transition 持久化。
- **内容**：Astro Content Collections，`src/content.config.ts` 定义 `blog/`、`projects/`、`wander/` 三个集合（markdown）。
- **评论 / 友链申请**：Giscus（GitHub Discussions）。友链自动同步：`src/data/links.json` 由 `scripts/check-links.js` + `scripts/sync-friends.js` 经 `.github/workflows/check-friends.yml`（每日北京时间 03:00）维护。
- **部署**：Vercel，push 到 `master` 自动部署；新工作走 `feature/*` 分支。
- **设计令牌**（`src/styles/global.css`）：日间 bg `#f5f0e8` / 夜间 `#0a0908`，accent `#8b7355` / `#c4a87a`，字体 Noto Serif SC（标题）/ Noto Sans SC（正文），SVG fractal-noise 纸纹，罗马数字 TOC + 点线引导，❧ fleuron 花饰。
- **既有页面**（罗马数字 i–vi）：about、work、blog（index + `[...slug]`）、wander、archive、friends，外加首页 index + 404 撕页。
- **既有组件**：TableOfContents、BottomNav、ThemeToggle、VinylPlayer、Giscus、InkCanvas（手写笔触 canvas）。

> **两处已知 GOTCHA**（贯穿多项功能）：
> 1. **TableOfContents.astro 与 BottomNav.astro 的导航 `items` 数组是手动重复的**——改一处必须同步改另一处，否则顶/底导航错位。
> 2. **罗马数字 i–vi 与导航顺序手动绑定**——新增导航项需手填编号，且首页 `index.astro` 的 `.toc-list li:nth-child` 入场动画延迟当前**只定义到 nth-child(6)**，新增第 7 项需补 `:nth-child(7)` 延迟。
> 3. **`astro.config.mjs` 当前为空 `defineConfig({})`，未设 `site` 字段**——多项功能（SEO、OG、元数据）的绝对 URL 生成均依赖它，是硬前置条件。

## 实施顺序建议

**核心思路**：先打地基（域名 + 元数据真源），再做依赖地基的派生能力，最后做改动面大的内容/双语工程。

1. **`site-meta`（先行，地基）**——它建立 `src/data/siteMeta.ts` 单一真源并要求在 `astro.config.mjs` 设 `site`。`seo-foundation` 与 `og-cards` 都依赖这个 `site` 字段与元数据，先做能避免重复配置和返工。**先向站主拍板：生产域名、社交 URL、spine 标语文案。**
2. **`seo-foundation`**——复用 `site-meta` 配好的 `site` 字段，加 sitemap/RSS/robots/canonical/OG。两者 OG 注入逻辑有重叠，建议合并实现（见下方备注）。
3. **`og-cards`**——在 SEO 的 OG meta 基础上，把静态 `og:image` 升级为动态书页卡。依赖 `site` 与本地字体。
4. **`now-page`（可随时插入）**——零依赖、工作量 S，可作为热身或并行小任务。唯一需注意第 7 个导航项的罗马数字与首页动画延迟。
5. **`work-alpha-cases`**——需在能访问 BRAIN MCP 的 Agent 会话里先烘焙 `alphas.json`，再做页面。改动集中、不阻断构建。
6. **`case-studies-demo`**——需本地 Flutter 构建产物，体积较大，建议独立分支。
7. **`i18n`（最后，改动面最大）**——几乎触及所有页面与导航组件，建议在前述功能稳定后再做，避免与它们反复冲突。

> **备注（重要）**：`site-meta`、`seo-foundation`、`og-cards` 三者都要往 `Base.astro` 的 `<head>` 注入 OG / canonical / meta，且都依赖 `astro.config` 的 `site`。**强烈建议把三者的 head 注入逻辑统一在一次 `Base.astro` 改造中完成**，由 `siteMeta.ts` 提供数据、`seo-foundation` 提供 sitemap/RSS/robots、`og-cards` 提供动态图，避免三次反复改同一段 `<head>`。

## 1. `now-page` · /now 夹页卡（本月在做什么）

> **档位**：立刻做 ｜ **工作量**：S

### 目标
新增 `/now` 路由页面（遵循 nownownow.com 约定），回答「这个月我在做什么 / 戴哪顶帽子」。内容来自单一 markdown 源文件，带「最后更新」日期，视觉上呈现为一张夹在书里的便签卡（slip-card），与现有书籍/编辑风格一致。

### 价值
站主同时是 AI 训练师、WorldQuant 量化研究员、Flutter 开发者，多个身份并行。`/now` 给访客（潜在合作方、招聘方、朋友）一个「当下快照」：现在重心在哪、戴哪顶帽子、近期可联系的事。维护成本极低——只改一个 markdown 文件、更新一个日期，比写整篇博客轻量得多，符合「慢就是快 / 低摩擦持续输出」的习惯。

### 依赖
- 无新增 npm 包。
- 现有 Astro 6.1.9 + `astro:content`（Content Collections / glob loader）。
- 现有组件：`Base.astro`、`ThemeToggle.astro`、`BottomNav.astro`。
- 现有设计令牌与样式：`global.css`（`.content-page` / `.prose` / `.page-enter` / `fadeUp` / CSS 变量）。
- 现有 `Base.astro` 的 `ClientRouter` 与 `astro:before-swap` / `astro:after-swap` 主题与动画钩子（无需新增钩子）。

### 改动文件

| 文件 | 动作 | 说明 |
|------|------|------|
| `src/content.config.ts` | 修改 | 新增 `now` collection（glob loader + zod schema：title/updated/location?/note?），并在 `collections` 导出对象追加 `now`。 |
| `src/content/now/current.md` | 新建 | 唯一的 /now 内容源文件：frontmatter + markdown 正文。日后唯一需手动维护的文件。 |
| `src/pages/now.astro` | 新建 | 新路由页面。`getCollection('now')` + `render()`，渲染正文为 `.prose` 并展示「最后更新」日期戳；套用 content-page + page-top + h1/subtitle + BottomNav，外加 slip-card 视觉。 |
| `src/components/TableOfContents.astro` | 修改 | items 数组追加 `{ label: "现在", href: "/now", page: "vii" }`。 |
| `src/components/BottomNav.astro` | 修改 | 同步追加同一条目（两处数组重复，必须同改）。 |

### 数据模型
新增 Content Collection `now`：

```ts
const now = defineCollection({
  loader: glob({ pattern: "**/*.md", base: "./src/content/now" }),
  schema: z.object({
    title: z.string(),               // 如「现在」
    updated: z.date(),               // 最后更新日期，必填，YYYY-MM-DD 裸值，由 z.date() 解析
    location: z.string().optional(), // 可选，如「上海」
    note: z.string().optional(),     // 可选，一句话状态副标题
  }),
});
export const collections = { blog, projects, wander, now };
```

`src/content/now/current.md` frontmatter 示例（约定只放一个 current.md）：

```markdown
---
title: "现在"
updated: 2026-05-28
location: "上海"
note: "这个月主要戴 AI 训练师这顶帽子"
---
## 正在做
- ...
```

页面取单条：`const entries = await getCollection("now"); const now = entries[0];`

### 实现步骤
1. 编辑 `src/content.config.ts`：在 `wander` 定义之后新增上述 `now` collection，末行改为 `export const collections = { blog, projects, wander, now };`（`glob` 与 `z` 已在顶部 import）。
2. 新建 `src/content/now/current.md`，写入 frontmatter（`updated` 写成不带引号的 `YYYY-MM-DD`，与 `blog/first-post.md` 的 `date: 2026-04-28` 一致）与正文（建议分节 `## 正在做` / `## 戴哪顶帽子` / `## 想聊的`）。
3. 新建 `src/pages/now.astro` frontmatter：`import Base / ThemeToggle / BottomNav`、`import { getCollection, render } from "astro:content"`，`const now = (await getCollection("now"))[0]; const { Content } = await render(now);`（参考 `blog/[...slug].astro`，但本页静态单页、无 `getStaticPaths`）。
4. 模板：`<Base title="现在" description="我现在在做什么">` 内放 `<article class="content-page">`，复用 page-top（`<a href="/" class="back-link">← 返回</a>` + `<ThemeToggle />`）、`<h1>现在</h1>`、`<p class="subtitle">{now.data.note ?? "这个月在做什么"}</p>`。
5. subtitle 下加 slip-card 容器：`<div class="now-card page-enter">` 内含 `<div class="prose"><Content /></div>`，底部一行更新戳：`最后更新 · {now.data.updated.toLocaleDateString("zh-CN", { year:"numeric", month:"2-digit", day:"2-digit" })}{now.data.location ? ` · ${now.data.location}` : ""}`（`toLocaleDateString` 写法照搬 `blog/index.astro`）。文末 `<BottomNav />`。
6. 写 scoped `<style>`：`.now-card` 用 `background: var(--card-bg)`、`border: 1px solid var(--border)`、`border-radius: 4px`、`padding: 1.8rem`、`box-shadow: 0 2px 14px rgba(0,0,0,0.06)`、左侧 `border-left: 3px solid var(--accent)` 当书签条；`.now-stamp` 用 `font-size:0.85rem; color: var(--muted); margin-top:1.5rem; border-top:1px solid var(--border); font-style: italic`。**所有颜色一律走 CSS 变量，禁止硬编码**，保证暗色自动适配。
7. `.now-card` 加 `page-enter` 复用 `fadeUp` 入场动画；可选 `transform: rotate(-0.4deg)` 轻微倾斜，hover 复正（transition 0.3s）。
8. 编辑 `TableOfContents.astro`：items 末尾追加 `{ label: "现在", href: "/now", page: "vii" }`。
9. 编辑 `BottomNav.astro`：做完全相同的追加（两处数组重复，漏改导致导航不一致）。
10. 本地验证：`npm run dev` 访问 /now 确认正文与更新戳；切暗色确认配色随变量；从首页点击确认 cross-fade 与 `page-enter`；`npm run build` 确认无 schema/类型报错。
11. 在 `feature/now-page` 提交，PR/preview 验证后再合 master。

### 边界情况
- `src/content/now/` 为空或缺 `current.md`：`entries[0]` 为 `undefined`，render 报错。**首版随 PR 一起提交 current.md**；可在取值后加 `if (!now)` 兜底占位文案。
- now 目录存在多个 md：`getCollection` 不排序，`entries[0]` 顺序不保证。约定只放一个；若未来保留历史快照，应按 `updated` 倒序 sort 后取首项。
- `updated` 写成带引号字符串或非法日期：`z.date()` 解析失败导致 build 报错；必须是裸 `YYYY-MM-DD`。
- 导航编号 `vii` 与现有 i–vi：罗马数字手动维护，TOC 与 BottomNav 两处都要改。
- 首页 `.toc-list li:nth-child` 动画延迟只到 nth-child(6)：第 7 项无错峰入场动画；如在首页 TOC 生效，需在 `index.astro` 的 `<style>` 补 `:nth-child(7)` 延迟。
- View Transitions / 直接进入 /now：纯静态、无 window 全局状态，`page-enter` 由 `Base.astro` 已有 `_animInit` 统一处理，刷新同页也正常。
- 暗色 FOUC：完全依赖 `Base.astro` 现有 `astro:before-swap` 预写 `data-theme` 逻辑，无需额外处理。

### 验收标准
- [ ] 访问 /now 正确渲染 `current.md` 正文（`.prose`）与「最后更新 · 日期」戳。
- [ ] 页面结构沿用 content-page + page-top（返回链接 + ThemeToggle）+ h1/subtitle + BottomNav，与 about/work/blog 视觉一致。
- [ ] 内容呈现为夹页便签卡（border + card-bg + 左侧书签条 + 轻微阴影/倾斜），颜色全走 CSS 变量，暗色自动适配。
- [ ] 顶部 TOC 与底部 BottomNav 都新增「现在」项且 `href=/now`、罗马数字一致。
- [ ] 从首页点进 /now 有 cross-fade、卡片有 fadeUp 入场；刷新 /now 正常显示。
- [ ] 只需编辑 `current.md` 的正文和 `updated` 即可更新页面，无需改 `.astro` 代码。
- [ ] `npm run build` 通过，无 zod / 类型 / 渲染错误。

- 是否将来接入自动更新（类似 friends 的 Action）？当前定位纯手动低摩擦，默认不接。

## 2. `site-meta` · 站点元数据中枢 + spine 标语 + 结构化数据

> **档位**：立刻做 ｜ **工作量**：M

### 目标
建立 `src/data/siteMeta.ts` 单一真源，集中管理站点标题、标语、描述、四顶帽子身份与社交链接；由 `Base.astro` 的 `<head>` 统一消费（title / description / Open Graph / canonical / Person JSON-LD with sameAs）。同时确定一句中英双语 spine 标语，把「AI 训练师 / 量化研究员 / Flutter 开发者 / 创作者」四重身份串成一句话，在首页与 about 呈现。

### 价值
现在元数据散落且不一致：BottomNav 硬编码社交链接，`Base.astro` 默认描述是泛泛的「个人博客」，无 OG、无 canonical、无 schema.org。本功能让站主只在一个文件改一次，全站 head + 社交链接 + 结构化数据同步；Person JSON-LD + sameAs 让搜索引擎把 GitHub、WorldQuant、社交账号识别为同一个人，提升个人品牌可发现性；一句 spine 标语让访客一眼看懂「四顶帽子」的人在做什么。

### 依赖
- 无新增 npm 包（手写 `<script type="application/ld+json">`）。
- 需在 `astro.config.mjs` 配置 `site` 字段（生成绝对 URL 必需）——当前为空。
- 依赖站主拍板 spine 标语文案与最终社交账号 URL。

### 改动文件

| 文件 | 动作 | 说明 |
|------|------|------|
| `src/data/siteMeta.ts` | 新建 | 元数据单一真源。导出 typed `siteMeta` + `buildPersonJsonLd(siteUrl)`。 |
| `astro.config.mjs` | 修改 | `defineConfig` 加 `site: 'https://<生产域名>'`，使 `Astro.site` 可用。 |
| `src/layouts/Base.astro` | 修改 | 导入 siteMeta；默认 description 引用 `siteMeta.description`；head 注入 title 模式、OG、Twitter Card、canonical、Person JSON-LD（`set:html`）。 |
| `src/components/BottomNav.astro` | 修改 | 删硬编码 socials，改从 `siteMeta.social` 派生（保留各自 inline SVG icon 映射）；渲染结构与 email-btn 逻辑不变。 |
| `src/pages/index.astro` | 修改 | `.identity` 硬编码「开发者 / 创作者」改为 `siteMeta.spineTaglineShort`；hour-mark 脚本不变。 |
| `src/pages/about.astro` | 修改 | `.subtitle` 或 prose 顶部加入 CN spine 长标语（可选附 EN）。 |

### 数据模型
```ts
export interface SocialLink {
  label: string;   // "GitHub" / "WorldQuant BRAIN" / "Twitter" / "Instagram" / "电子邮件"
  href: string;    // 完整 URL 或 mailto:
  sameAs: boolean; // 是否纳入 JSON-LD sameAs（邮箱通常 false）
}
export interface SiteMeta {
  siteName: string;          // "Freetrip"
  brandName: string;         // "夜 / siyuan"
  url: string;               // 生产域名，与 astro.config site 一致
  description: string;       // 默认 meta description
  spineTaglineCN: string;    // 长版中文标语
  spineTaglineEN: string;    // 长版英文标语
  spineTaglineShort: string; // 首页短版「AI 训练师 · 量化研究员 · Flutter 开发者 · 创作者」
  ogImage?: string;          // 缺省回退 favicon.svg
  locale: string;            // "zh_CN"
  author: {
    name: string;            // schema.org Person.name
    alternateName: string;   // "夜 / Freetrip"
    jobTitle: string[];      // ["AI Trainer","Quant Researcher","Flutter Developer","Writer"]
  };
```

## 3. `seo-foundation` · SEO 基建：sitemap + RSS + robots + canonical + 全站 meta/OG

## 4. `og-cards` · 动态书页式 OG 分享卡

> **档位**：立刻做 ｜ **工作量**：M

### 目标
为每条路由 / 每篇文章在构建期自动生成 1200×630 社交分享图，视觉做成一页摊开书页：顶部「Freetrip · 第一卷」页眉、右上罗马数字页码、右下翻起折角（dog-ear）、❧ 花饰与纸纹底色，中央标题/描述。再在 `Base.astro` head 注入 `og:image` / `twitter:image` 指向这些静态 PNG。

### 依赖

### 数据模型
无新增集合。两个 TS 结构：
```ts
type OgRoute = { slug: string; title: string; subtitle?: string; page: string /* 罗马数字, 对齐 TOC i–vi */ };
const STATIC_OG_ROUTES: OgRoute[] = [
  { slug: 'home',    title: 'Freetrip',  subtitle: '开发者 / 创作者', page: '' },
```

### 实现步骤
1. `astro.config.mjs` 的 `defineConfig` 加 `site`（待站主确认域名）。生成绝对 `og:image` 的前提。
2. `npm i satori @resvg/resvg-js`（可选 `satori-html`）。三者纯 Node/wasm，Vercel 默认 `astro build` 可跑，不需 adapter，保持静态。

4. 新建 `og-template.ts`，`renderOgPng(opts)`。版式（1200×630）复刻设计令牌：底色 `#f5f0e8`（**OG 卡固定走日间色**，规避暗色分享图偏暗）、文字 `#2c2c2c`、强调 `#8b7355`；左上 kicker `Freetrip · 第一卷`（Serif、`letter-spacing 0.15em`，对齐 `.volume-label`）；右上大号罗马数字 page（斜体淡 muted）；中央标题（Serif 700 大字号）+ 下方 subtitle（Sans）；标题旁 ❧ 花饰；右下绝对定位三角 + 阴影画 dog-ear；外层书页边距 + 细 hairline 模拟书脊。`satori(vnode, { width:1200, height:630, fonts })` 得 SVG，`new Resvg(svg).render().asPng()` 得 PNG，返回 `Uint8Array`。

### 边界情况
- 超长标题/描述：satori 不自动截断，需 `og-template.ts` 做字数硬截断 + 省略号或多行 clamp，否则溢出。
- CJK 子集缺字：生僻字/emoji 不在子集会留空/方块；要么全量字体（体积大），要么补字。
- blog 嵌套子目录（`src/content/blog/foo/bar.md` → id `foo/bar`）：slug 变 `blog/foo/bar`，产物 `/og/blog/foo/bar.png` 合法，但 rest 参数与 `.png` 拼接不能丢层级；Base 默认推导也用同规则。
- 集合记录无 description：subtitle 回退空或固定副标题，不能渲染 `undefined`。
- date 缺省：本卡不强依赖，若版式想显示日期需处理 undefined。
- `Astro.site` 未配置：`new URL(rel, undefined)` 抛错/生成相对地址；步骤 1 必须先做并兜底告警。
- resvg/satori 在 Vercel 构建期原生二进制兼容：本地 win32 与 CI linux 二进制不同，靠 npm `optionalDependencies` 自动选包，**提交时不要锁单平台**。
- 404 页无对应 OG 路由：`resolveOgRoute` 命不中回退首页卡 `/og/home.png`。
- 构建期 png 数量随文章线性增加（单图数十到上百 ms）：文章很多时拉长 build，可接受；量大时可只为 blog 生成、固定页用预渲染 png。

## 7. `i18n` · 中英双语 i18n（Astro 内建 + 边注式第二语言）

> **档位**：值得做 ｜ **工作量**：L

### 目标
为这个 Astro 静态站接入官方 i18n，提供可被搜索引擎抓取的 `/zh`（根路径）与 `/en` 两套 URL；UI 文案集中到字典按语言渲染；ClientRouter 切页与刷新时保持当前语言（cookie 存储）；并用「书页边注（marginalia）」方式在内容页展示第二语言，而非全量复制每篇文章。

### 价值
站主是 CN/EN 双语作者，读者跨语言。现在全站 UI 文案（导航「关于/作品集/想法…」、按钮「返回」「切换主题」「第一卷」等）与标题全硬编码中文。接入后：英文读者拿到可抓取的 `/en/*` 链接（利于分享与 SEO），切语言后浏览同一篇内容不被打断，且站主不必为每篇随笔写两遍正文——只在需要处补一段边注译文，符合「翻开的书」气质。

### 依赖
- **无新增 npm 包**（Astro 6.1.9 内建 `astro:i18n` 与 i18n routing 足够）。
- 沿用现有 `ClientRouter`，不引入框架。
- 沿用现有 window-global + `astro:before-swap`/`after-swap` 状态持久化模式。
- Giscus（评论 mapping 可能受 `/en` 前缀影响，见待定问题）。

## 约定

### 分支策略
- 遵循 README 约定：**新工作一律走 `feature/*` 分支**，每个功能一条分支（如 `feature/now-page`、`feature/site-meta`、`feature/seo-foundation`、`feature/og-cards`、`feature/work-alpha-cases`、`feature/case-studies-demo`、`feature/i18n`）。
- **不直接在 `master` 上开发。** 合并前先在 PR / Vercel preview 验证。

### 提交风格
- 匹配现有中文 Conventional Commits 习惯，例如：
  - `feat: 新增 /now 夹页卡页面`
  - `feat: 建立 siteMeta 元数据中枢与 Person 结构化数据`
  - `feat: 接入 sitemap/RSS/robots 与全站 OG`
  - `fix: 修正 OG 卡长标题溢出截断`
  - `chore: 烘焙 BRAIN alpha 数据快照到 alphas.json`
- 类型用语：`feat`（全新功能）、`fix`（修缺陷）、`chore`（构建/数据/配置）、`docs`（文档）、`style`（样式）。

### 部署提醒
- **push 到 `master` 会触发 Vercel 自动部署**。务必先在 feature 分支与 preview 验证，确认 `npm run build` 通过、线上无 404、跨页 View Transition 与主题/语言切换无回归后，再合并 `master`。
- 涉及 `astro.config.mjs` 的 `site` 字段、`public/` 下大体积产物（Flutter web、字体、OG 图）的改动，合并前务必确认 Vercel 构建环境能正确解析与托管。
